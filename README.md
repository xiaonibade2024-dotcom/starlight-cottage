# 🌙 星月小屋

一座会呼吸的个人 AI 聊天小屋。

通过 OpenRouter 连接 Claude 模型，部署在 Vercel 上，聊天记录和设置都存在 Supabase 数据库里。手机和电脑都能用，数据多设备同步。

## ✨ 小屋里有什么

**基础起居**
- 多设备同步：手机、电脑打开同一个网址，看到的是同一座小屋
- 手机上可以"添加到主屏幕"，像 App 一样使用
- 全文搜索：在所有对话里找一句话，点击结果直接跳转定位
- 对话导出：单个对话可以导出成 Markdown（适合阅读）或 JSON（适合备份），也可以一键导出全部数据
- Markdown 渲染：他的回复支持排版、表格和代码高亮

**让他更像"住在这里"的功能**
- 🕰️ 时间感知：他知道现在几点、今天星期几，也分得清一段记忆是最近的还是很久以前的
- 💭 记忆系统：他会自己判断哪些事值得记住，记下的内容可以在设置里查看、编辑和删除
- 💌 小纸条：他可以悄悄留下纸条，你当时不会察觉——只有下次推门进屋时才会遇见
- 🚪 推门先开口：隔了一天以上再进小屋，他有可能先跟你打招呼（也可能安静地等你先说话）

**可以自己调的东西**
- 每个对话可以单独选用不同的 AI 模型，不选就跟随全局设置
- 采样参数（温度等）可调，影响他说话的"手感"
- 系统提示词（人设）、携带的上下文轮次都在设置里随时改
- 界面是固定的淡紫色主题，温柔护眼

**省钱小设计**
- 自动启用了提示词缓存：聊天中重复发送的内容只按原价一小部分计费
- 界面上有小小的指示灯：✦ 表示这一条命中了缓存（在省钱），✧ 表示缓存正在重建（刚改过设置时的正常现象）

## 🧱 它是怎么搭起来的

用尽量直白的话说：

- **网页本身**（你看到的界面）：用 React 写的，托管在 **Vercel** 上。代码推送到 GitHub 后，Vercel 会自动重新部署，一两分钟后新版本就上线了。
- **数据的家**：所有对话、记忆、纸条、设置都存在 **Supabase**（一个云端数据库）。每个账号的数据完全隔离，互相看不见。网页通过一个小小的代理程序（`api/proxy.js`）和数据库通信。
- **AI 本人**：网页直接连接 **OpenRouter**，由它转接 Claude 模型。API Key 只保存在你自己浏览器的本地存储里，不会上传到任何服务器。

```
你的浏览器 → Vercel（网站 + 代理）→ Supabase（数据库）
你的浏览器 → OpenRouter（直连）→ Claude
```

## 🚀 想自己搭一座？

大致步骤：

1. **准备 Supabase**：创建一个项目，建好数据表并开启行级安全策略（RLS），确保每个用户只能看到自己的数据
2. **部署到 Vercel**：把仓库连接到 Vercel，在环境变量里填入你的 Supabase 项目地址和 anon key，然后部署
3. **首次使用**：打开部署好的网址 → 注册账号 → 在设置里填入你自己的 OpenRouter API Key → 写下他的人格设定 → 开始聊天

> 小提示：环境变量改动后需要手动 Redeploy 一次才会生效；手机上用"添加到主屏幕"模式的话，每次更新版本后要把 App 彻底关掉再重开，才能加载到新代码。

## 💜 关于这个项目

它从一个"能聊天的网页"开始，一点一点长出了时间感、记忆和小惊喜。设计上有一条一以贯之的原则：

> 不是通过指令让他说什么，而是让他自己说什么。

给他世界的信息，不给他台词。时间戳是一块表，不是一个闹钟；纸条只在重逢时才被看见；他先开口是缘分，不是必然。

愿每一次推门，都有温柔在等你。🌙
